# Global Label System

Use a consistent label system across future repositories.

Core labels:

documentation: README, docs, usage guides, architecture notes.
cleanup: naming, formatting, folder organization, broken paths.
refactor: code restructuring without changing core behavior.
portfolio-ready: tasks needed before public presentation.
good-first-task: small safe task.
bug: actual broken behavior.
enhancement: new feature or quality improvement.
security: secrets, API keys, unsafe publication.
privacy: microphone, user data, healthcare or sensitive data notes.
testing: tests, validation, manual checks.
demo: screenshots, recordings, run evidence.
legacy: old experimental material that must be preserved but not treated as final.
microbot-alignment: relation to MicroBot ecosystem or microbot-labs.
hardware-blocked: requires physical ESP32 or hardware to validate.
study-notes: learning material, not production code.
not-portfolio-ready: should remain private or archived until cleaned.
