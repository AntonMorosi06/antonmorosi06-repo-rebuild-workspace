# Known limitations

This is an educational genetic algorithm simulation, not a production optimizer.

The rockets follow fixed DNA vectors and do not use sensors or real-time decision-making.

The fitness function is intentionally simple. It rewards closeness, completion and speed, but it does not model energy efficiency, smoothness, control stability or safety constraints.

The obstacle system currently supports a simple rectangular obstacle. More complex environments can be added later.

The simulation does not currently export generation data to CSV or JSON.

The project intentionally avoids claiming hardware validation, robotics validation or recovered historical source equivalence.
