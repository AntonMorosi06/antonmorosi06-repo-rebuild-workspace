# Master Rebuild Strategy

This workspace organizes the analyzed repositories into coherent project families.

The main strategy is:

1. Keep MicroBot-related repositories separate but aligned.
2. Do not merge legacy repositories blindly into microbot-labs.
3. Convert strong experimental repositories into clean portfolio projects.
4. Keep rough study repositories private unless transformed into clean learning labs.
5. Separate public-ready projects from private archives.
6. Add screenshots, README files, known limitations and usage instructions before publication.
7. Avoid publishing secrets, private data, generated environments, broken notebooks, or placeholder claims.
8. Use GitHub Issues and labels to turn cleanup into visible engineering work.

2. microbot20-legacy-simulation-lab
